# 软著说明书格式规范（默认）

## 一、封面格式

| 元素 | 字体 | 字号 | 加粗 | 对齐 | 其他 |
|------|------|------|------|------|------|
| 软件名称 | 黑体 | 22pt | 是 | 居中 | - |
| 文档类型（用户手册） | 黑体 | 22pt | 是 | 居中 | - |
| 著作权人 | 宋体 | 16pt | 是 | 居中 | 在页面底部最后两行 |
| 日期 | 宋体 | 16pt | 是 | 居中 | 在页面底部最后两行 |

**封面布局**：
- 软件名称和文档类型在页面上部居中
- 著作权人和日期在页面底部最后两行
- 使用空行分隔，确保落款在页面底部

## 二、页眉格式

| 位置 | 内容 | 字体 | 字号 | 对齐 |
|------|------|------|------|------|
| 左侧 | 软件名称 + 版本号 | 宋体 | 9pt | 左对齐 |
| 右侧 | 页码 | - | - | 使用多个空格推到右侧 |

**页眉实现方式**：
```
软件名称 V1.0                              [页码]
```
- 左侧：软件名称 + 空格 + 版本号
- 使用多个空格将页码推到右侧（不使用制表位，制表位可能不生效）
- 页码使用PAGE域代码

## 六、文字颜色

- 所有文字颜色均为黑色（RGB: 0, 0, 0）

## 七、写作规范

### 内容组织
- 按功能模块组织，面向用户操作
- 描述操作步骤，而不是系统功能
- 截图配文字说明，图文结合
- 语言简洁明了，面向普通用户
- **每个功能模块必须有"注意事项"**

### 章节结构
```
一、平台总体介绍（必须有，第一章）
  （一）平台功能概述
  （二）总体架构
  （三）技术实现
  （四）数据流向
二、系统登录
  （操作描述 + 截图 + 注意事项）
三、xxx功能模块
  （一）xxx子功能
    正文描述...
    图X xxx示意
    注意事项：
    1. xxx
    2. xxx
```

### 正文写作示例
```
在系统首页，点击右上角新建项目按钮，弹出新建项目对话框。如下图。

图3 新建项目按钮

在对话框中填写项目名称和项目描述，点击确定按钮，即可创建新项目。如下图。

图4 新建项目对话框

注意事项：
1. 项目名称不能重复，如果输入已存在的名称，系统会提示错误。
2. 项目名称长度不超过50个字符。
3. 项目描述长度不超过500个字符。
```
