# 迭代质量优化流程

## 问题背景

首轮生成的内容通常不达标：
- 总字数不足（8000-10000字 < 15000字）
- 部分章节字数不足（<1500字）
- 部分节字数不足（<300字）
- 配图不足

## 解决方案：迭代循环

```
┌─────────────────────────────────────────────────────────────┐
│ 第1轮：生成                                                  │
│  1. 🤖 模型按章节分步生成JSON内容                            │
│  2. 🐍 脚本检查每章基础质量（review_chapter.py）              │
│  3. 🐍 脚本验证整体内容质量（validate_content_quality.py）    │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 第2轮：修改（如果验证不通过）                                 │
│  4. 🤖 模型根据验证报告补充内容                               │
│     - 字数不足：补充详细描述                                  │
│     - 配图不足：补充图片标记                                  │
│  5. 🐍 脚本再次验证内容质量                                   │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 第N轮：重复修改直到验证通过                                   │
│  6. 🐍 脚本合并所有章节（merge_chapters.py）                  │
│  7. 🐍 生成架构图（excalidraw skill）                         │
└─────────────────────────────────────────────────────────────┘
```

## 使用 iterate_content.py

```bash
# 验证内容质量并获取修改建议
python3 scripts/iterate_content.py --content content.json
```

输出示例：

## 修改策略

| 问题类型 | 修改方式 |
|----------|----------|
| 字数不足 | 补充详细描述，增加操作步骤 |
| 配图不足 | 在关键操作处添加"如下图"和图片标记 |
| 章节不足 | 拆分或合并章节，保持6-10章 |

## 典型迭代次数

- 第1轮：生成基础内容（~8000字）
- 第2轮：补充细节（~12000字）
- 第3轮：补充配图和注意事项（~15000字）
- 第4轮：微调达标

## 与 TDD 的关系

迭代质量优化是 TDD 在文档生成中的应用：
- **Red**: 验证不通过 → 发现问题
- **Green**: 修改内容 → 解决问题
- **Refactor**: 优化质量 → 达标
