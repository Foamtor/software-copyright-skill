     3|## 问题背景
     4|
     5|Mermaid和matplotlib生成的配图都存在不足：
     6|
     7|**Mermaid问题**：
     8|- 图片尺寸不适合Word文档插入
     9|- 字体太小，打印后可读性差
    10|- 自动布局不适合A4纸排版
    11|- 需要额外安装mermaid-cli
    12|
    13|**matplotlib问题**（用户反馈：2026-06-04）：
    14|- 样式不符合传统软件图风格（默认是数据可视化风格）
    15|- 文字在打印时仍然偏小
    16|- 插入Word时图片尺寸控制不够精确
    17|- 缺少专业的软件架构图元素

   115|**问题**：缺少中文字体
   116|**解决**：
   117|```bash
   118|# 安装中文字体
   119|sudo apt install fonts-wqy-microhei
   120|# 或使用系统自带字体
   121|```
   122|
   123|### Q3: 图片在Word中太小
   124|**问题**：插入后需要手动调整
   125|**解决**：在Word中设置图片宽度为15cm（A4纸宽度的80%）
   126|
   127|### Q4: 图片模糊
   128|**问题**：分辨率太低
   129|**解决**：设置dpi=300或更高
   130|```python
   131|fig.savefig('output.png', dpi=300)
   132|```

   151|
   152|### 基础架构图
   153|
   154|```python
   155|import matplotlib.pyplot as plt
   156|from matplotlib.patches import FancyBboxPatch
   157|
   158|fig, ax = plt.subplots(figsize=(10, 7.5))
   159|
   160|# 绘制层级
   161|layers = [
   162|    ('表示层', '#2E86AB', 0.8),
   163|    ('业务层', '#A23B72', 0.6),
   164|    ('数据层', '#2D6A4F', 0.4),
   165|]
   166|
   167|for name, color, y in layers:
   168|    rect = FancyBboxPatch((0.1, y-0.08), 0.8, 0.15,
   169|                         boxstyle="round,pad=0.02",
   170|                         facecolor=color, alpha=0.3,
   171|                         edgecolor=color, linewidth=2)
   172|    ax.add_patch(rect)
   173|    ax.text(0.5, y, name, ha='center', va='center', fontsize=14)
   174|
   175|ax.set_xlim(0, 1)
   176|ax.set_ylim(0, 1)
   177|ax.axis('off')
   178|plt.tight_layout()
   179|plt.savefig('architecture.png', dpi=300, bbox_inches='tight')
   180|```

   182|## 替换现有docx中的图片
   183|
   184|当需要替换已生成docx中的图片时，使用zipfile操作：
   185|
   186|```python
   187|import zipfile
   188|import shutil
   189|
   190|def replace_images_in_docx(docx_path, output_path, image_mapping):
   191|    """
   192|    替换docx文件中的图片
   193|    
   194|    image_mapping: {原文件名: 新图片路径}
   195|        例如: {'image1.png': '/path/to/new_image.png'}
   196|    """
   197|    temp_dir = "/tmp/docx_temp"
   198|    os.makedirs(temp_dir, exist_ok=True)
   199|    
   200|    # 解压
   201|    with zipfile.ZipFile(docx_path, 'r') as zip_ref:
   202|        zip_ref.extractall(temp_dir)
   203|    
   204|    # 替换图片（直接覆盖，不要创建.backup文件）
   205|    for original_name, new_image_path in image_mapping.items():
   206|        original_path = os.path.join(temp_dir, 'word', 'media', original_name)
   207|        if os.path.exists(original_path):
   208|            shutil.copy2(new_image_path, original_path)
   209|    
   210|    # ⚠️ 重要：删除任何.backup文件，否则会被打包进docx增加文件大小
   211|    media_dir = os.path.join(temp_dir, 'word', 'media')
   212|    for filename in os.listdir(media_dir):
   213|        if filename.endswith('.backup'):
   214|            os.remove(os.path.join(media_dir, filename))
   215|    
   216|    # 重新打包
   217|    with zipfile.ZipFile(output_path, 'w', zipfile.ZIP_DEFLATED) as zipf:
   218|        for root, dirs, files in os.walk(temp_dir):
   219|            for file in files:
   220|                file_path = os.path.join(root, file)
   221|                arcname = os.path.relpath(file_path, temp_dir)
   222|                zipf.write(file_path, arcname)
   223|    
   224|    shutil.rmtree(temp_dir)
   225|```
   226|
   227|### 常见陷阱
   228|
   229|1. **备份文件膨胀**：如果在替换时创建 `.backup` 文件，它们会被打包进docx，导致文件大小翻倍。替换完成后务必清理。
   230|2. **图片映射顺序**：docx中图片按 `image1.png`, `image2.png` 等顺序存储，需要确认哪个编号对应哪张图。
   231|3. **中文字体**：WSL环境下 `WenQuanYi Zen Hei` 比 `SimHei` 更可靠（后者可能未安装）。

   234|
   235|**配图方案优先级**：
   236|1. **Excalidraw Diagram Skill** — 专业、美观、内置验证，最适合编码agent

   241|**关键教训**（2026-06-04）：
   242|- 用户对图表样式要求高，需要"符合传统软件图风格"
   243|- matplotlib默认样式不符合专业文档要求
   244|- 图片插入Word时必须精确控制尺寸
   245|- 文字必须足够大，打印后仍清晰可读
