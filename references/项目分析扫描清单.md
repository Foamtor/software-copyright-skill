# 项目分析扫描清单（Phase 1 高效扫描）

Phase 1 的目标是快速理解项目的功能模块、技术栈和业务逻辑。以下是按技术栈分类的扫描清单，按优先级排序。

## 通用扫描顺序

```
1. README.md / package.json / pyproject.toml / requirements.txt  → 项目名称、版本、依赖
2. 路由文件 → 功能模块清单（前端路由 = 用户可见功能）
3. 主入口文件 → API路由注册、中间件、全局配置
4. 页面/视图文件 → 每个功能模块的具体操作
5. 数据模型/Schema → 业务实体定义
6. 配置文件 → 功能开关、工具列表、环境参数
7. Prompt模板 → AI能力边界和业务场景
```

## Vue.js 项目扫描清单

### 必读文件（按优先级）

| # | 文件 | 提取信息 |
|---|------|----------|
| 1 | `package.json` | 项目名称、版本、依赖库（UI、地图、图表） |
| 2 | `src/router/config.js` 或 `src/router/index.js` | **路由表 → 功能模块清单**（最重要） |
| 3 | `src/views/` 下每个页面 `.vue` | 每个功能模块的具体操作和交互 |
| 4 | `src/components/` 下公共组件 | 共享功能（地图、对话、图表） |
| 5 | `src/services/` 或 `src/api/` | API调用接口 |
| 6 | `src/utils/` | 工具函数（地图投影、加密等） |
| 7 | `src/main.js` | 全局注册的插件和组件 |

### Vue.js 扫描技巧

- 路由文件的 `path` 和 `component` 映射 = 功能模块到页面的映射
- `meta` 字段可能包含权限、标题等元信息
- `views/` 目录结构通常反映功能模块分组
- `components/` 中的 `.vue` 文件行数可快速判断复杂度：`wc -l *.vue | sort -n`

## FastAPI 项目扫描清单

### 必读文件（按优先级）

| # | 文件 | 提取信息 |
|---|------|----------|
| 1 | `requirements.txt` / `pyproject.toml` | 依赖库（FastAPI、asyncpg、OpenAI等） |
| 2 | `main.py` | **API路由注册 → 功能模块清单**（最重要） |
| 3 | `*/controller.py` | 每个模块的API端点定义 |
| 4 | `*/models.py` | 请求/响应数据模型（Pydantic） |
| 5 | `config.py` | 配置项：LLM模型、数据库、功能开关、工具列表 |
| 6 | `prompt/*.md` | Prompt模板 → AI能力边界和业务场景 |
| 7 | `*/orchestrator.py` 或 `*/service.py` | 业务编排逻辑 |

### FastAPI 扫描技巧

- `app.include_router(router, prefix="/xxx")` → 模块到URL前缀的映射
- `APIRouter(prefix="/xxx", tags=["xxx"])` → 每个controller的路由前缀
- `@router.get/post("/path")` → 具体API端点
- Pydantic `BaseModel` 定义 = 请求/响应的数据结构
- `config.py` 中的 `TOOL_LIST` = Agent的工具清单（非常重要的业务信息）

## 通用扫描命令

```bash
# 前端路由文件
find src -name 'config.js' -path '*/router/*' -o -name 'index.js' -path '*/router/*'

# 前端页面组件（按行数排序）
find src/views -name '*.vue' | xargs wc -l | sort -n

# 后端controller文件
find . -name 'controller.py' -not -path '*/.venv/*' -not -path '*/__pycache__/*'

# 后端模型文件
find . -name 'models.py' -not -path '*/.venv/*' -not -path '*/__pycache__/*'

# 后端prompt模板
find prompt -type f | head -30

# 后端配置文件
find . -name 'config.py' -not -path '*/.venv/*'

# 排除第三方库统计代码行数
find . -name '*.py' -not -path '*/.venv/*' -not -path '*/__pycache__/*' | xargs wc -l | tail -1
find src -name '*.vue' -o -name '*.js' -o -name '*.ts' | xargs wc -l | tail -1
```

## 扫描结果输出格式

扫描完成后，按以下结构整理信息供后续阶段使用：

```

1. 首页 (/) — 功能描述
2. 模块A (/module-a) — 功能描述
...

后端功能模块：
1. agents — Agent核心引擎
2. ai_map_chat — 地图AI对话
...

技术栈：
- 前端: Vue.js 2.x + Element UI + OpenLayers + ECharts
- 后端: Python + FastAPI + PostgreSQL + asyncpg
- AI: mimo-v2.5 / qwen3.6-plus
- 向量: BGE-M3
```

## 常见陷阱

1. **不要扫描 `.venv/`、`node_modules/`、`__pycache__/`** — 始终在 find 命令中排除
2. **Windows路径** — WSL下前端路径可能是 `/mnt/d/...`，后端也一样
3. **Vue 2 vs Vue 3** — 从 `package.json` 的 `vue` 版本号判断，扫描方式相同但 Composition API 写法不同
4. **多个 config 文件** — 后端可能有 `config.py`、`runtime_settings.json`、`.env` 等多个配置源
5. **代码行数统计** — 排除第三方库后统计，前端排除 `node_modules/`，后端排除 `.venv/`
