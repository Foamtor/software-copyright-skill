# 软著申请文档要求（标准版）

## 一、信息采集表

两个表格：

### 表1：软件基本信息（19行8列）

| 序号 | 字段 | 要求 |
|------|------|------|
| 1 | 软件全称 | 必须以"系统、软件、平台"结尾 |
| 2 | 软件简称 | 字数少于全称 |
| 3 | 版本号 | 如 V1.0 |
| 4 | 软件分类 | 应用软件/嵌入式软件/中间件/操作系统 |
| 5 | 作品说明 | 原创/修改（修改需50字说明） |
| 6 | 开发方式 | 独立/合作/委托/下达任务 |
| 7 | 著作权人 | 姓名、国籍、城市、类别、证件 |
| 8 | 完成日期 | 不早于著作权人成立日期 |
| 9 | 发表状态 | 未发表/已发表 |

### 表2：开发与运行环境（15行4列）

| 字段 | 字数限制 |
|------|----------|
| 开发硬件环境 | ≤50字 |
| 运行硬件环境 | ≤50字 |
| 开发操作系统 | ≤50字 |
| 开发工具 | ≤50字 |
| 运行平台 | ≤50字 |
| 运行支撑环境 | ≤50字 |
| 编程语言 | 无限制 |
| 源程序量 | 行数 |
| 开发目的 | 8-50字 |
| 面向领域 | 4-50字 |
| 主要功能 | 100-200字 |
| 软件属性 | 最多3项 |
| 技术特点 | ≤100字 |

## 二、文档（二选一）

### 使用说明书
- 登录页面、操作步骤、截图+描述
- 截图连贯，体现页面跳转和数据变化

### 设计说明
- 软件结构图、流程图、逻辑框图
- 总体设计、接口设计、模块/函数功能、算法、运行设计

### 页数要求
- ≥60页：前30页+后30页
- <60页：全部提交
- 每页≥30行（图片除外）

## 三、源代码文档

- 每页≥50行
- 前30页+后30页，共60页
- 不足60页全部提交

## 四、页眉格式

- 左上角：软件名称+版本号
- 右上角：页码
