# 软著说明书写作规范

## 核心原则

**软著说明书是用户操作指南，不是技术文档！**

## 正确写法 vs 错误写法

### ✅ 正确：面向用户操作

```
一、系统登录

双击桌面快捷方式或从开始菜单启动农业调查数据治理分析软件，系统显示登录界面。如下图。
图1 登录界面

在登录界面输入用户名和密码，点击"登录"按钮，即可进入系统首页。首次登录请使用管理员分配的账号密码。
```

```
二、项目管理
（一）新建项目

在系统首页，点击右上角"新建项目"按钮，弹出新建项目对话框。如下图。
图3 新建项目按钮

在对话框中填写项目名称和项目描述，点击"确定"按钮，即可创建新项目。如下图。
图4 新建项目对话框

项目创建成功后，自动刷新项目列表，新项目显示在列表中。
```

### ❌ 错误：技术架构风格

```
一、项目管理功能：支持创建和管理农业调查数据治理项目，实现数据治理工作的规范化管理，
提供项目状态监控和版本控制能力。

二、数据导入功能：支持CSV、Excel、JSON、Shapefile等多种农业调查数据格式的智能导入，
提供自动识别数据结构、字段映射、数据验证和实时预览功能。
```

这种写法像功能清单，不像操作指南。

## 写作要点

1. **描述用户动作**：点击什么按钮、填写什么内容、选择什么选项
2. **描述系统响应**：弹出什么对话框、显示什么页面、提示什么信息
3. **操作顺序**：按用户操作的先后顺序组织
4. **截图配合**：每段操作后跟"如下图。"，然后是"图X xxx示意"
5. **语言简洁**：面向普通用户，不使用技术术语
6. **每段独立**：每个操作步骤是一个独立的段落

## 章节组织

```
一、系统登录
  （操作描述 + 截图）

二、项目管理
  （一）新建项目
    （操作描述 + 截图）
  （二）项目列表
    （操作描述 + 截图）
  （三）项目详情
    （操作描述 + 截图）

三、数据导入
  （一）文件上传
    （操作描述 + 截图）
  （二）数据预览
  （三）选择模板
  （四）字段映射
  （五）导入数据

四、数据治理
  （一）数据清洗
  （二）数据校验
  （三）脚本执行

五、数据资源
六、统计分析
七、数据导出
```
