# 内容生成策略

## 问题
首轮生成的内容通常只有8000-10000字，距离15000字要求有较大差距。

## 解决策略：迭代补充

### 第1轮：生成基础内容
- 每章生成2-3个section
- 每个section的content约400-500字
- 预期总字数：8000-10000字

### 第2轮：补充细节
- 对每章的section进行补充
- 增加操作步骤的详细描述
- 增加更多的注意事项
- 预期增加：2000-3000字

### 第3轮：深度补充
- 对字数不足的section进行深度补充
- 增加更多的截图标记
- 增加更多的注意事项
- 预期增加：1000-2000字

## 补充技巧

### 1. 增加操作步骤描述
```
修改前：用户点击"登录"按钮即可完成登录。
修改后：用户在手机号码输入框中输入注册时使用的手机号码，在密码输入框中输入账户密码，
然后点击"登录"按钮即可完成登录。如果输入的手机号码或密码错误，系统会提示
"手机号码或密码错误，请重新输入"。
```

### 2. 增加注意事项
```
修改前：
注意事项：
1. 项目名称不能为空

修改后：
注意事项：
1. 项目名称不能为空，且长度不超过50个字符
2. 项目创建后不可删除，请谨慎操作
3. 如果项目名称已存在，系统会提示修改
4. 建议使用有意义的项目名称，方便后续查找
```

### 3. 增加截图标记
```
修改前：用户点击"登录"按钮即可完成登录。

修改后：用户点击"登录"按钮即可完成登录。如下图。
图21 登录按钮
```
