# 模板驱动架构设计（docxtpl方案）

## 核心思路

模型只输出JSON内容，格式100%由Word模板控制。

```
┌─────────────┐     ┌─────────────────┐     ┌─────────────┐
│  LLM 生成   │ ──→ │   content.json  │ ──→ │ docxtpl 渲染│
│  JSON 内容   │     │  (结构化数据)    │     │  生成.docx  │
└─────────────┘     └─────────────────┘     └─────────────┘
                                                  ↑
                                          templates/*.docx
                                          (格式由模板控制)
```

## 为什么选docxtpl

| 特性 | docxtpl | 自己写python-docx |
|------|---------|-------------------|
| 格式保持 | ✅ 100%（模板决定） | ❌ 需要逐个设置 |
| 页眉页脚 | ✅ 自动保留 | ❌ 需要手动处理 |
| 表格样式 | ✅ 自动保留 | ❌ 需要手动设置 |
| 图片插入 | ✅ 支持 | ❌ 需要自己写 |
| 循环/条件 | ✅ Jinja2语法 | ❌ 需要自己实现 |
| 维护成本 | 低 | 高 |

**GitHub**: `elapouya/python-docx-template` | **Stars**: ~1500+
**安装**: `pip install docxtpl`

## 模板中的Jinja2语法

```
{{cover_title}}                          ← 变量
{% for section in sections %}            ← 循环
  {{section.heading}}
  {{section.content}}
  {% for img in section.images %}
    {{img.ref}} {{img.description}}
  {% endfor %}
  {% for note in section.notes %}
    注意：{{note}}
  {% endfor %}
{% endfor %}
```

## 处理格式变化

```
用户参考文档 ──→ create_template.py ──→ 模板.docx
                                              ↓
                                      模型输出JSON
                                              ↓
                                      generate_manual.py
                                              ↓
                                         最终.docx
```

格式变化 = 换模板，不需要改代码。

## ⚠️ 关键教训：内容质量 ≠ 格式质量

**问题**：模板驱动解决了格式问题，但最初生成的内容只有2013字（旧版18864字）。

**根因**：
- 格式问题：模板控制 ✅ 已解决
- 内容问题：取决于JSON质量 ⚠️ 需要单独保障

**解决方案**：在SKILL.md中加入内容质量规范：
- 总字数 ≥15000字
- 截图标记 ≥50张
- 注意事项 ≥30个
- 每章 ≥1500字
- 每节 ≥300字

**教训**：架构改进不等于内容改进，两者需要分别保障。
