# 常见问题与解决方案

## 1. 格式验证问题

**问题**：生成的文档字号可能不符合要求（如Heading1显示14pt而非16pt）

**解决**：生成后用Python检查实际字号：
```python
from docx import Document
doc = Document('说明书.docx')
for para in doc.paragraphs[:10]:
    if para.style.name.startswith('Heading'):
        print(f"{para.style.name}: {para.runs[0].font.size}")
```

如字号不对，用脚本修正：
```python
from docx.shared import Pt
for para in doc.paragraphs:
    if para.style.name == 'Heading 1':
        for run in para.runs:
            run.font.size = Pt(16)
```

## 2. 图片标题与实际图片不匹配

**问题**：文档中有图片标题（如"图5 登录界面"）但没有实际图片

**说明**：这是正常的，图片标题是占位符。用户需要手动补充系统截图，在提交清单中说明哪些图片需要补充。

## 3. .doc格式文件读取

**问题**：python-docx只支持.docx，不支持旧版.doc格式

**解决**：使用olefile库读取.doc文件：
```python
import olefile
ole = olefile.OleFileIO('文件.doc')
stream = ole.openstream('WordDocument')
data = stream.read()
text = data.decode('utf-16-le', errors='ignore')
```

## 4. 占位符跨多个run

**问题**：Word中占位符可能被拆分到多个run中，导致替换失败

**解决**：fill_template.py已处理此情况，先尝试单run替换，失败则合并处理。

## 5. 中文字体设置

**问题**：python-docx设置字体后，中文可能显示为默认字体

**解决**：必须同时设置 `w:eastAsia` 属性：
```python
run.font.name = '黑体'
run._element.rPr.rFonts.set(qn('w:eastAsia'), '黑体')
```

## 6. 页眉页码

**问题**：需要动态页码，不能硬编码

**解决**：使用域代码（PAGE field）：
```python
fldChar1 = OxmlElement('w:fldChar')
fldChar1.set(qn('w:fldCharType'), 'begin')
instrText = OxmlElement('w:instrText')
instrText.text = ' PAGE '
fldChar2 = OxmlElement('w:fldChar')
fldChar2.set(qn('w:fldCharType'), 'end')
```

## 7. 表格合并单元格

**问题**：python-docx处理合并单元格较复杂

**解决**：替换时检查cell是否被合并（`_tc` 是否有 `gridSpan` 属性）。

## 8. 配图字体太小

**问题**：matplotlib默认字体在打印时太小

**解决**：
```python
import matplotlib.pyplot as plt
plt.rcParams['font.sans-serif'] = ['WenQuanYi Zen Hei', 'SimHei', 'DejaVu Sans']
plt.rcParams['font.size'] = 14  # 正文字号
plt.rcParams['axes.titlesize'] = 18  # 标题字号
```

## 9. 代码文档行数不足

**问题**：提取的代码行数不够60页

**解决**：按65页（3250行）提取，预留封面等额外页数。如果项目代码量不足，可以适当包含配置文件和SQL文件。

## 10. JSON解析失败

**问题**：中文引号（「」）导致json.loads失败

**解决**：生成JSON时避免使用中文引号，改用【】或直接省略。如果必须用，在Python中用 `\u201c` `\u201d` 转义。
