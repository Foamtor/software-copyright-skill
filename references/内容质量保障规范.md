# 内容质量保障规范

## 问题背景

模板驱动架构解决了格式问题，但内容质量仍然取决于模型生成的JSON。

**对比数据**：
- 旧版（模型直接生成）：18864字，66张截图，39个注意事项
- 新版（手写JSON）：2013字，40张截图，15个注意事项
- 新版（加入质量规范后）：6527字，40张截图，~15个注意事项

## 内容质量指标

| 指标 | 标准 | 检查方法 |
|------|------|----------|
| **总字数** | ≥15000字 | `sum(len(p.text) for p in doc.paragraphs if p.text.strip())` |
| **章节数** | 6-10章 | 按功能模块组织 |
| **截图标记** | ≥50张 | `count([截图: xxx])` 或 `count(图X)` |
| **注意事项** | ≥30个 | `count(注意事项)` |
| **每章字数** | ≥1500字 | 详细描述操作步骤 |
| **每节字数** | ≥300字 | 包含完整操作流程 |

## 写作风格要求

**软著说明书是用户操作指南，不是技术文档！**

### 正确写法
```
在系统首页，点击"新建项目"按钮，弹出新建项目对话框。如下图。
图3 新建项目按钮

在对话框中填写项目名称和项目描述，点击"确定"按钮，即可创建新项目。如下图。
图4 新建项目对话框

注意事项：
1. 项目名称不能为空，且长度不超过50个字符
2. 项目创建后不可删除，请谨慎操作
```

### 错误写法
```
一、项目管理功能：支持创建和管理项目，提供项目状态监控和版本控制能力。
```

### 写作要点

1. **面向用户操作**：描述用户点击什么按钮、填写什么内容、看到什么结果
2. **按功能模块组织**：一、二、三大章，（一）（二）小章，1. 2. 3. 三级
3. **截图配文字**：每段操作描述后跟"如下图。"，然后是"图X xxx示意"
4. **语言简洁**：面向普通用户，不使用技术术语
5. **图文结合**：截图位置用"图X xxx"标记，居中显示
6. **操作步骤**：按用户操作顺序组织，不是按系统功能模块
7. **每个功能必须有"注意事项"**：提醒用户常见问题和操作要点
8. **必须有"平台总体介绍"章节**：第一章必须是平台总体介绍

## JSON内容质量要求

生成content.json时，必须确保：

1. **每个section的content字段**：≥300字，详细描述操作步骤
2. **每个section的notes字段**：≥3个注意事项
3. **每个section的images字段**：≥2张截图标记
4. **每个subsection的content字段**：≥200字
5. **每个subsection的notes字段**：≥2个注意事项

## 自动质量检查脚本

```python
from docx import Document

def check_content_quality(docx_path):
    """检查内容质量"""
    doc = Document(docx_path)
    
    total_chars = sum(len(p.text) for p in doc.paragraphs if p.text.strip())
    screenshot_count = sum(1 for p in doc.paragraphs if '[截图' in p.text or p.text.strip().startswith('图'))
    note_count = sum(1 for p in doc.paragraphs if '注意事项' in p.text)
    chapter_count = sum(1 for p in doc.paragraphs if p.text.strip().startswith(('一、', '二、', '三、', '四、', '五、', '六、', '七、', '八、', '九、', '十、')))
    
    print(f"=== 内容质量检查 ===")
    print(f"总字数: {total_chars} (要求≥15000) {'✅' if total_chars >= 15000 else '❌'}")
    print(f"章节数: {chapter_count} (要求6-10章) {'✅' if 6 <= chapter_count <= 10 else '❌'}")
    print(f"截图标记: {screenshot_count} (要求≥50) {'✅' if screenshot_count >= 50 else '❌'}")
    print(f"注意事项: {note_count} (要求≥30) {'✅' if note_count >= 30 else '❌'}")
    
    return total_chars >= 15000 and 6 <= chapter_count <= 10 and screenshot_count >= 50 and note_count >= 30
```

## 教训总结

1. **架构改进 ≠ 内容改进**：模板驱动解决了格式，但内容质量需要单独保障
2. **质量指标要量化**：字数、截图、注意事项都要有明确的数字标准
3. **写作风格要明确**：正确/错误示例对比，让模型知道"好"的标准
4. **JSON示例很重要**：给模型一个高质量的JSON示例作为参考
5. **自动检查不可少**：生成后必须验证内容质量是否达标
