     1|# 截图功能规范
     2|
     3|## 概述
     4|
     5|软著说明书需要大量系统截图，本功能提供两种截图模式：
     6|1. **模式A：自动截图** - 给URL，自动访问系统并截图
     7|2. **模式B：智能匹配** - 用户截图放到文件夹，自动匹配到说明书章节
     8|
     9|## 模式B：智能匹配（推荐）
    10|
    11|### 使用场景
    12|
    13|- 系统不方便自动化访问（需要VPN、内网等）
    14|- 系统登录复杂（多因素认证、短信验证等）
    15|- 用户已经有很多截图，想快速整理
    16|- 需要精确控制截图内容
    17|
    18|### Agent驱动工作流程（推荐）
    19|
    20|在Hermes Agent环境中，推荐以下工作流程：
    21|
    22|1. **用户截图** → 放到文件夹（如 `/path/to/screenshots/`）
    23|2. **Agent生成说明书内容JSON** → 使用 `[截图: xxx]` 标记
    24|3. **Agent逐张分析截图** → 使用 `vision_analyze` 工具识别页面内容
    25|4. **Agent智能匹配** → 根据分析结果匹配到说明书章节
    26|5. **Agent生成匹配报告** → 供用户审阅确认
    27|6. **Agent插入截图** → 使用 `generate_manual.py --images-dir`
    28|
    29|**关键**：`match_screenshots.py` 脚本是辅助工具，实际的视觉分析由agent通过 `vision_analyze` 工具完成。脚本负责文件扫描和报告生成，agent负责调用视觉模型分析每张截图。
    30|
    31|### 使用流程
    32|
    33|#### 1. 准备截图
    34|
    35|将系统截图放到一个文件夹中，建议命名规范：
    36|```
    37|screenshots/
    38|├── 01_登录页面.png
    39|├── 02_系统首页.png
    40|├── 03_用户管理.png
    41|├── 04_数据查询.png
    42|└── ...
    43|```
    44|
    45|命名建议：
    46|- 使用数字前缀表示顺序
    47|- 包含功能名称
    48|- 使用中文命名更直观
    49|
    50|#### 2. 生成说明书内容
    51|
    52|先按正常流程生成说明书内容JSON：
    53|```json
    54|{
    55|  "一、系统概述": {
    56|    "（一）系统简介": "本系统是...",
    57|    "（二）功能概览": "系统主要功能包括..."
    58|  },
    59|  "二、用户登录": {
    60|    "（一）登录入口": "在浏览器中输入系统地址，进入登录页面。如下图。\n[截图: 登录页面]",
    61|    "（二）登录流程": "输入用户名和密码，点击登录按钮。如下图。\n[截图: 登录表单]"
    62|  }
    63|}
    64|```
    65|
    66|#### 3. 运行匹配脚本
    67|
    68|```bash
    69|python3 scripts/match_screenshots.py \
    70|  --screenshots /path/to/screenshots \
    71|  --manual /path/to/manual_content.json \
    72|  --output /path/to/match_report.json
    73|```
    74|
    75|#### 4. 审阅匹配结果
    76|
    77|匹配报告会生成两个文件：
    78|- `match_report.json` - JSON格式，供脚本使用
    79|- `match_report_可读.md` - Markdown格式，方便审阅
    80|
    81|示例报告：
    82|```markdown
    83|# 截图匹配报告
    84|
    85|共匹配 5 张截图
    86|
    87|## 匹配 1
    88|
    89|- **截图**: 01_登录页面.png
    90|- **目标章节**: 登录入口
    91|- **章节路径**: 二、用户登录/（一）登录入口
    92|- **匹配度**: 0.85
    93|- **置信度**: high
    94|```
    95|
    96|#### 5. 确认并插入
    97|
    98|审阅后确认匹配结果，然后运行插入脚本：
    99|```bash
   100|python3 scripts/insert_screenshots.py \
   101|  --manual /path/to/manual.docx \
   102|  --matches /path/to/match_report.json \
   103|  --screenshots /path/to/screenshots \
   104|  --output /path/to/manual_with_screenshots.docx
   105|```
   106|
   107|### 匹配算法
   108|
   109|匹配基于以下因素：
   110|1. **页面标题匹配** - 截图标题与章节标题的相似度
   111|2. **关键词匹配** - 功能关键词的重合度
   112|3. **页面类型匹配** - 登录页、列表页、表单页等类型的对应关系
   113|
   114|匹配阈值默认0.3，可通过 `--threshold` 参数调整。
   115|
   116|### 注意事项
   117|
   118|1. **截图质量**
   119|   - 分辨率建议1920x1080或更高
   120|   - 避免截取浏览器边框
   121|   - 确保页面内容清晰可读
   122|
   123|2. **命名规范**
   124|   - 文件名包含功能关键词
   125|   - 避免使用特殊字符
   126|   - 使用中文命名更直观
   127|
   128|3. **视觉模型**
   129|   - 匹配需要视觉模型支持
   130|   - 当前配置：mimo-v2.5（有视觉能力）
   131|   - 如需切换：`请根据你的AI Agent配置视觉模型`
   132|
   133|## 模式A：自动截图
   134|
   135|### 使用场景
   136|
   137|- 系统可以自动化访问
   138|- 需要批量截图多个页面
   139|- 截图内容相对固定
   140|
   141|### 使用流程
   142|
   143|#### 1. 准备登录信息
   144|
   145|**方式一：Cookie文件**
   146|```json
   147|[
   148|  {
   149|    "name": "session_id",
   150|    "value": "abc123",
   151|    "domain": "example.com"
   152|  }
   153|]
   154|```
   155|
   156|**方式二：账号密码**
   157|```bash
   158|python3 scripts/auto_screenshot.py \
   159|  --url http://example.com \
   160|  --username admin \
   161|  --password 123456 \
   162|  --output /path/to/screenshots
   163|```
   164|
   165|#### 2. 定义功能模块（可选）
   166|
   167|创建模块配置文件 `modules.json`：
   168|```json
   169|[
   170|  {
   171|    "name": "登录页面",
   172|    "path": "/login",
   173|    "description": "系统登录入口"
   174|  },
   175|  {
   176|    "name": "系统首页",
   177|    "path": "/",
   178|    "description": "登录后的首页"
   179|  },
   180|  {
   181|    "name": "用户管理",
   182|    "path": "/admin/users",
   183|    "description": "用户列表和管理"
   184|  }
   185|]
   186|```
   187|
   188|#### 3. 运行自动截图
   189|
   190|```bash
   191|# 按模块截图
   192|python3 scripts/auto_screenshot.py \
   193|  --url http://example.com \
   194|  --cookie cookies.json \
   195|  --modules modules.json \
   196|  --output /path/to/screenshots
   197|
   198|# 根据说明书自动截图
   199|python3 scripts/auto_screenshot.py \
   200|  --url http://example.com \
   201|  --cookie cookies.json \
   202|  --manual manual_content.json \
   203|  --output /path/to/screenshots
   204|```
   205|
   206|#### 4. 查看结果
   207|
   208|截图保存到输出目录，同时生成 `screenshot_results.json` 记录截图信息。
   209|
   210|### 登录处理
   211|
   212|#### Cookie登录
   213|1. 在浏览器中手动登录系统
   214|2. 使用浏览器开发者工具导出cookie
   215|3. 保存为JSON文件
   216|4. 运行脚本时指定 `--cookie` 参数
   217|
   218|#### 账号密码登录
   219|1. 确定用户名和密码输入框的选择器
   220|2. 确定登录按钮的选择器
   221|3. 运行脚本时指定相应参数
   222|
   223|常见选择器：
   224|- 用户名：`input[name="username"]`, `#username`, `.login-input`
   225|- 密码：`input[name="password"]`, `#password`, `.password-input`
   226|- 登录按钮：`button[type="submit"]`, `.login-btn`, `#login-button`
   227|
   228|### 注意事项
   229|
   230|1. **页面加载**
   231|   - 脚本会等待页面加载完成
   232|   - 可通过 `--wait-time` 调整等待时间
   233|   - 某些SPA应用需要更长等待
   234|
   235|2. **动态内容**
   236|   - 某些内容通过JavaScript动态加载
   237|   - 可能需要额外等待或触发操作
   238|   - 建议使用 `--visible` 模式调试
   239|
   240|3. **反爬虫**
   241|   - 某些系统有反爬虫机制
   242|   - 可能需要设置User-Agent、添加延迟
   243|   - Cookie登录通常更可靠
   244|
   245|4. **截图质量**
   246|   - 默认1920x1080分辨率
   247|   - 可通过 `--full-page` 截取整页
   248|   - 避免截取浏览器边框和工具栏
   249|
   250|## 集成到工作流程
   251|
   252|### 在说明书中使用截图
   253|
   254|#### 方式一：占位符标记
   255|在说明书中使用 `[截图: xxx]` 标记：
   256|```json
   257|{
   258|  "（一）登录入口": "在浏览器中输入系统地址，进入登录页面。如下图。\n[截图: 登录页面]"
   259|}
   260|```
   261|
   262|#### 方式二：图片文件名
   263|直接引用图片文件名：
   264|```json
   265|{
   266|  "（一）登录入口": "在浏览器中输入系统地址，进入登录页面。如下图。\n图1 登录页面"
   267|}
   268|```
   269|
   270|### 与generate_manual.py集成
   271|
   272|`generate_manual.py` 脚本支持自动插入截图：
   273|```bash
   274|python3 scripts/generate_manual.py \
   275|  --content chapter_content.json \
   276|  --images-dir /path/to/screenshots \
   277|  --output chapter.docx
   278|```
   279|
   280|脚本会：
   281|1. 读取内容JSON中的截图标记
   282|2. 从截图目录中查找对应图片
   283|3. 插入到Word文档的正确位置
   284|4. 添加图片说明（图X xxx示意）

   286|## 常见问题
   287|
   288|### Q1: 匹配不准确怎么办？
   289|**A**: 
   290|- 检查截图文件名是否包含功能关键词
   291|- 调整匹配阈值（`--threshold` 参数）
   292|- 手动修正匹配报告中的映射关系
   293|
   294|### Q2: 截图太多怎么筛选？
   295|**A**: 
   296|- 按功能模块分子文件夹存放
   297|- 只选择关键页面截图
   298|- 每个功能1-2张截图即可
   299|
   300|### Q3: 自动截图登录失败？
   301|**A**: 
   302|- 尝试使用cookie方式登录
   303|- 检查选择器是否正确
   304|- 使用 `--visible` 模式观察登录过程
   305|- 可能需要处理验证码等额外步骤
   306|
   307|### Q4: 截图插入后格式不对？
   308|**A**: 
   309|- 检查图片分辨率（建议300dpi以上）
   310|- 确认Word文档的图片宽度设置
   311|- 使用 `replace_docx_images.py` 调整图片尺寸
