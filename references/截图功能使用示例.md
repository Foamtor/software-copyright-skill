   140|## 场景：自动截图模式
   141|
   142|如果系统可以自动化访问，可以使用自动截图模式：
   143|
   144|### 步骤1：准备cookie
   145|
   146|在浏览器中登录系统，导出cookie为JSON文件：
   147|
   148|```json
   149|[
   150|  {
   151|    "name": "JSESSIONID",
   152|    "value": "ABC123XYZ",
   153|    "domain": "example.com"
   154|  },
   155|  {
   156|    "name": "token",
   157|    "value": "eyJhbG...VCJ9...",
   158|    "domain": "example.com"
   159|  }
   160|]
   161|```
   162|
   163|保存为 `cookies.json`。
   164|
   165|### 步骤2：运行自动截图
   166|
   167|```bash
   168|python3 scripts/auto_screenshot.py \
   169|  --url http://example.com \
   170|  --cookie cookies.json \
   171|  --manual /path/to/manual_content.json \
   172|  --output /path/to/screenshots/auto
   173|```
   174|
   175|脚本会：
   176|1. 使用cookie登录系统
   177|2. 根据说明书内容中的功能模块
   178|3. 自动导航到各个页面并截图
   179|4. 保存截图到输出目录
   180|
   181|### 步骤3：后续处理
   182|

   185|## 常见问题
   186|
   187|### Q1: 匹配不准确怎么办？
   188|
   189|**A**: 
   190|1. 检查截图文件名是否包含功能关键词
   191|2. 调整匹配阈值：`--threshold 0.5`（提高阈值，减少误匹配）
   192|3. 手动修正 `match_report.json` 中的映射关系
   193|
   194|### Q2: 截图太多怎么筛选？
   195|
   196|**A**:
   197|1. 按功能模块分子文件夹存放
   198|2. 只选择关键页面截图（每个功能1-2张）
   199|3. 使用 `--threshold` 参数过滤低匹配度的结果
   200|
   201|### Q3: 自动截图登录失败？
   202|
   203|**A**:
   204|1. 尝试使用cookie方式登录
   205|2. 检查选择器是否正确
   206|3. 使用 `--visible` 模式观察登录过程
   207|4. 可能需要处理验证码等额外步骤
   208|
   209|### Q4: 截图插入后格式不对？
   210|
   211|**A**:
   212|1. 检查图片分辨率（建议300dpi以上）
   213|2. 确认Word文档的图片宽度设置
   214|3. 使用 `replace_docx_images.py` 调整图片尺寸
