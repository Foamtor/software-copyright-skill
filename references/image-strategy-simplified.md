# 配图策略（精简版）

## 核心原则

1. **必要性** — 只配必要的图，不配装饰性图
2. **代表性** — 每个功能模块配1张代表性截图
3. **关键节点** — 只在关键操作节点配图

## 配图数量标准

| 指标 | 原标准 | 精简标准 | 说明 |
|------|--------|----------|------|
| 总配图数 | 50-80张 | **13-16张** | 减少75% |
| 每章配图 | 5-10张 | **2-3张** | 减少70% |
| 架构图 | 5张 | **1张** | 只保留总体架构 |
| 功能截图 | 40+张 | **12-15张** | 每模块1-2张 |

## 配图类型

| 类型 | 数量 | 来源 | 说明 |
|------|------|------|------|
| 系统总体架构图 | 1张 | 绘制 (excalidraw) | 展示系统整体设计 |
| 功能模块截图 | 12-15张 | 截图 (实际系统) | 每模块1-2张代表性界面 |
| **总计** | **13-16张** | - | 精简但完整 |

## 各章配图建议

| 章节 | 必须配图 | 可选配图 |
|------|----------|----------|
| 第一章：总体介绍 | 架构图、首页截图 | 功能入口截图 |
| 第二章：登录与首页 | 登录页面、首页布局 | 搜索功能 |
| 第三章：展览展会 | 展会列表、展会详情 | 展品详情 |
| 第四章：帮扶服务 | 帮扶列表、帮扶申请 | 帮扶记录 |
| 第五章：AI顾问 | AI对话页面、聊天历史 | 推荐内容 |
| 第六章：地图查询 | 地图页面、位置搜索 | 导航功能 |
| 第七章：个人中心 | 个人中心、设置页面 | 收藏管理 |

## 配图位置规则

```
✅ 正确：正文描述 → "如下图。" → 图片 → 图名
❌ 错误：图片列表、图片在图名下方、图片与正文分离
```

## 配图命名规则

```
图{序号} {描述}
示例：图1 系统总体架构图、图2 首页整体布局
```

## 配图来源决策树

```
图片需求
├── 系统架构/流程 → 绘制（excalidraw）
│   └── 系统总体架构图（1张）
│
└── 界面展示 → 截图（实际系统）
    ├── 首页整体截图（1张）
    ├── 各功能模块主界面（6张）
    └── 核心操作流程（5-8张）
```
