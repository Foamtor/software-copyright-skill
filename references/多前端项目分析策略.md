# 多前端项目识别与分析策略

## 典型项目结构模式

现代项目可能同时存在多个前端实现，常见模式：

```
project_root/
├── ui_desgin/       # 功能设计原型（React/Vite/TailwindCSS）
│   ├── src/App.tsx  # 单文件包含所有页面和功能定义
│   └── src/data.ts  # 数据模型定义
├── agent_app/       # 移动端APP代码（React Native/Expo）
│   ├── app/app/     # 文件式路由
│   └── app/src/     # features/domain/infra 分层
├── agent_ui/        # Web端代码（Vue.js/Element UI）
│   └── src/views/   # 页面组件
└── agent_core/      # 后端代码（Python/FastAPI）
    └── main.py      # API入口
```

## 分析策略

### 1. 确认主次关系

| 目录 | 角色 | 用途 |
|------|------|------|
| `ui_desgin/` | **功能设计** | 软著说明书的功能模块以此为准 |
| `agent_app/` | **技术实现** | 提供技术架构、框架版本、代码行数 |
| `agent_ui/` | 辅助/历史 | 可能是早期Web端，非主前端 |
| `agent_core/` | **后端** | API接口、数据库、AI模型 |

### 2. 关键确认问题

首次项目分析时必须向用户确认：

> 项目有多个前端目录（ui_desgin / agent_app / agent_ui），请确认：
> 1. 功能模块以哪个为准？
> 2. 哪个是主要前端实现？

### 3. 信息提取分工

| 信息类型 | 来源 | 说明 |
|----------|------|------|
| 功能模块列表 | ui_desgin | Tab页、子页面、钻取页面 |
| 页面交互流程 | ui_desgin | 状态管理、页面跳转、筛选逻辑 |
| 技术栈/框架 | agent_app | React Native版本、依赖库 |
| 代码行数 | agent_app + agent_core | 前后端分别统计 |
| API接口 | agent_core | 后端路由、数据模型 |
| 数据模型 | ui_desgin/src/data.ts | 实体定义、字段结构 |

## 实际案例：国农臻汇APP

### 项目结构
- `ui_desgin/` - React 19 + Vite + TailwindCSS 4（App.tsx 5369行）
- `agent_app/` - React Native 0.81.5 (Expo 54) + TypeScript
- `agent_ui/` - Vue.js 2.x + Element UI（Web端，非主前端）
- `agent_core/` - Python FastAPI + PostgreSQL

### 功能模块（以ui_desgin为准）
1. **首页** - Banner轮播、搜索扫码、10大品类、5个一级标签钻取
2. **展览展会** - 日历视图、展会列表、展会回顾
3. **帮扶** - 帮扶产品瀑布流、乡村工匠列表
4. **AI顾问** - AI对话、预设问题、农品资讯
5. **我的** - 用户信息、统计、收藏、溯源日志

### 技术架构（以agent_app为准）
- React Native (Expo 54) + TypeScript
- Expo Router（文件式路由）
- Zustand + React Query
- 高德地图（expo-gaode-map）
- 224个源文件 + 22个路由文件

## 常见陷阱

1. **默认扫描第一个前端目录** → 可能扫描到Web端而非主APP
2. **功能设计和代码混为一谈** → 设计原型可能比代码更完整/更新
3. **忽略ui_desgin的数据模型** → data.ts包含完整的实体定义
4. **只看代码不看设计** → 软著说明书需要以用户视角的功能描述

## UI设计分析技巧

### 识别Tab导航

```javascript
// 搜索状态变量定义
grep -n 'useState.*Tab\|setCurrentTab\|activeTab' App.tsx

// 搜索底部导航栏
grep -n 'Bottom.*Nav\|navigation.*bar\|setCurrentTab' App.tsx
```

### 识别钻取页面

```javascript
// 搜索子页面状态
grep -n 'setActiveHomeSub\|activeHomeSub\|Drilldown' App.tsx
```

### 识别详情页

```javascript
// 搜索详情页状态
grep -n 'setSelected.*\|selected.*\|Detail\|detail' App.tsx
```

## 技术架构分析技巧

### React Native/Expo项目

```bash
# 查看路由结构
find app/app -name '*.tsx' | sort

# 查看features目录
find app/src/features -maxdepth 1 -type d

# 查看技术栈
cat app/package.json | grep -E '"dependencies"|"devDependencies"' -A 100
```

### Vue.js项目

```bash
# 查看路由配置
cat src/router/config.js

# 查看页面组件
find src/views -name '*.vue' | sort
```

## 截图工作流

1. **获取宿主机IP**: `ip route show default | awk '{print $3}'`
2. **访问UI设计**: `http://172.22.240.1:3000/`
3. **获取APP元素坐标**: `browser_console` + `getBoundingClientRect()`
4. **逐Tab截图**: `browser_console` 点击Tab → `browser_vision` 截图
5. **裁剪APP区域**: PIL裁剪到指定坐标
6. **验证截图**: `vision_analyze` 检查内容
