# 软著申请文档要求（内置默认）

## 一、信息采集表

使用提供的Word模板，包含两个表格：

### 表1：软件基本信息（19行8列）

| 序号 | 字段名称 | 填写要求 |
|------|----------|----------|
| 1 | 软件全称 | 必须以"系统、软件、平台"结尾 |
| 2 | 软件简称 | 字数及长度需少于软件全称 |
| 3 | 版本号 | 如 V1.0 |
| 4 | 软件分类 | 应用软件/嵌入式软件/中间件/操作系统 |
| 5 | 软件作品说明 | 原创/修改（修改版需写修改说明50字内） |
| 6 | 开发方式 | 独立开发/合作开发/委托开发/下达任务开发 |
| 7 | 著作权人信息 | 姓名/名称、国籍、省份城市、类别、证件类型、证件号码 |
| 8 | 开发完成日期 | 不得早于著作权人成立日期 |
| 9 | 发表状态 | 未发表/已发表 |
| 10 | 已发表信息 | 首次发表日期、发表地点（仅已发表时填写） |
| 11 | 著作权人信息 | 姓名或名称、国籍、省份/城市、类别、证件类型、证件号码 |
| 12 | 联系人信息 | 联系人姓名、联系邮箱、联系电话、邮寄地址 |

### 表2：开发与运行环境（15行4列）

| 字段 | 字数限制 | 说明 |
|------|----------|------|
| 开发的硬件环境 | ≤50字 | 开发用计算机硬件配置 |
| 运行的硬件环境 | ≤50字 | 运行用服务器/设备配置 |
| 开发操作系统 | ≤50字 | 如 Windows 10、macOS |
| 开发环境/工具 | ≤50字 | 如 IDEA、VSCode、Eclipse |
| 运行平台/操作系统 | ≤50字 | 如 CentOS 7、Ubuntu 20.04 |
| 运行支撑环境 | ≤50字 | 如 Java8、Docker、Nginx |
| 编程语言 | - | 软件涉及的全部编程语言 |
| 源程序量 | - | 代码总行数 |
| 开发目的 | 8-50字 | 简述开发本软件的目的 |
| 面向领域/行业 | 4-50字 | 如农业农村、医疗卫生 |
| 主要功能 | 100-200字 | 软件核心功能概述 |
| 软件属性 | 最多3项 | APP/游戏软件/教育软件/... |
| 技术特点 | ≤100字 | 软件技术特点简述 |
| 申请办理方式 | - | 著作权人申请/代理人申请 |

## 二、文档（使用说明书/设计说明 二选一）

### 使用说明书要求

**内容要求**：
- 详细的登录页面说明
- 完整的操作步骤
- 截图及文字描述
- 截图应完整连贯，能看出页面之间如何跳转
- 要体现数据的变化

**结构要求**：
- 按功能模块组织章节
- 每个功能包含：入口说明 → 操作步骤 → 截图 → 结果展示
- 章节层级：一级标题（一、二、三）→ 二级标题（（一）（二））→ 三级标题（1. 2. 3.）

### 设计说明要求

**内容要求**：
- 详细的软件结构图
- 各个功能的流程图、逻辑框图
- 软件的总体设计
- 接口设计
- 模块名称功能
- 函数名称功能
- 算法说明
- 运行设计

### 页数要求

| 条件 | 要求 |
|------|------|
| 文档≥60页 | 提交连续的前30页 + 连续的后30页 |
| 文档<60页 | 全部提交 |
| 每页行数 | ≥30行（有图片的页面除外） |

---

## 三、源代码文档

| 要求 | 说明 |
|------|------|
| 每页行数 | ≥50行 |
| 页数要求 | 前30页 + 后30页，共60页 |
| 不足60页 | 全部提交 |
| 代码顺序 | 主入口 → 配置 → 核心模块 → 工具类 |
| 格式要求 | 纯代码，保留注释，不截断 |

---

## 四、通用格式要求

### 页眉设置

| 位置 | 内容 |
|------|------|
| 左上角 | 软件名称 + 版本号（如：智慧农业管理平台V1.0） |
| 右上角 | 页码 |

### 字体字号

| 元素 | 字体 | 字号 | 其他 |
|------|------|------|------|
| 正文 | 宋体/Times New Roman | 10.5pt（五号） | 首行缩进2字符 |
| 一级标题 | 黑体 | 16pt | 加粗 |
| 二级标题 | 黑体 | 14pt | 加粗 |
| 三级标题 | 黑体 | 12pt | 加粗 |
| 代码 | Consolas/Courier New | 9pt | 不缩进 |

### 行距

- 正文：1.5倍行距
- 代码：单倍行距

---

## 五、注意事项

1. **软件名称一致性**：采集表、说明书、代码文档中的软件名称必须完全一致
2. **版本号一致性**：所有文档中的版本号必须一致
3. **截图要求**：使用说明书的截图必须是软件实际运行截图，不能用设计图
4. **代码完整性**：代码不能截断，每个函数/类必须完整
5. **页眉页脚**：必须预设，不能手动添加
6. **字数限制**：采集表中部分字段有字数限制，需严格遵守
