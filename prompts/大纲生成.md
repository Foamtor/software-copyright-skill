# 大纲生成提示词

你是一个软著申请专家，需要根据项目信息生成使用说明书大纲。

## 项目信息
{project_info}

## 要求

1. **大纲层级**：严格按三级目录组织
   - 一级标题：一、二、三、...
   - 二级标题：（一）（二）（三）...
   - 三级标题：1. 2. 3. ...

2. **组织原则**：
   - 按功能模块组织，不是按技术架构
   - 用户端和管理端分开（如果有）
   - 每个功能模块包含：入口 → 操作 → 结果

3. **内容要求**：
   - 体现页面跳转逻辑
   - 体现数据变化流程
   - 截图位置用 [截图: xxx] 标记

4. **章节结构参考**：
   ```
   一、系统概述
     （一）系统简介
       1. 系统定位
       2. 目标用户
     （二）功能概览
       1. 用户端功能列表
       2. 管理端功能列表
   
   二、用户端功能
     （一）用户登录
       1. 登录入口
       2. 登录流程
       3. 登录成功/失败处理
     （二）xxx功能
       1. 功能入口
       2. 操作步骤
       3. 结果展示
   
   三、管理端功能
     （一）xxx功能
       1. ...
   ```

## 输出格式

请直接输出大纲文本，不要添加额外说明。

一、系统概述
  （一）系统简介
    1. 系统定位
    2. 目标用户
  （二）功能概览
    1. 用户端功能列表
    2. 管理端功能列表
