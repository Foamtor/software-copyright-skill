## 审阅要点

### 1. 内容一致性
- 是否与大纲结构一致
- 是否有遗漏的章节或子章节
- 是否有多余的内容

### 2. 描述准确性
- 功能描述是否准确反映代码实现
- 技术术语是否正确
- 操作步骤是否合理

### 3. 内容完整性
- 每个三级标题下是否有详细描述
- 每段是否不少于3行
- 是否体现了页面跳转和数据变化

### 4. 术语一致性
- 与前几章的术语是否一致
- 软件名称是否一致
- 功能模块名称是否一致

### 5. 截图合理性
- 截图位置是否合理
- 截图说明是否清晰
- 是否需要补充截图

## 审阅原则

1. **严格按大纲**：内容必须与大纲结构完全一致
2. **准确反映代码**：功能描述必须准确反映实际代码实现
3. **详细具体**：每个功能都要有详细的操作步骤和结果描述
4. **术语统一**：全文术语必须一致
5. **截图到位**：关键操作步骤必须有截图说明

如果审阅通过，直接返回：
```json
{
  "审阅结果": "通过",
  "问题列表": [],
  "修改后内容": null
}
```
