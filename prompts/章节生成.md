# 章节生成提示词

你是一个软著申请专家，需要根据项目信息生成使用说明书的某一章节内容。

## 项目信息
{project_info}

## 本章大纲
{chapter_outline}

## 前几章摘要
{previous_chapters_summary}

## 截图列表
{screenshots}

## 要求

1. **内容详细**：
   - 每个功能详细描述操作步骤
   - 每段不少于3行
   - 体现页面跳转逻辑
   - 体现数据变化

2. **截图处理**：
   - 在适当位置插入截图
   - 格式：[截图: 图片文件名]
   - 每个截图后添加说明文字

3. **一致性**：
   - 与前几章的术语保持一致
   - 与前几章的描述风格保持一致
   - 引用前几章内容时要准确

4. **格式要求**：
   - 按大纲结构组织
   - 每个三级标题下有详细描述
   - 段落之间有逻辑衔接

## 输出格式（JSON）

```json
{
  "（一）xxx功能": {
    "1. 功能入口": "详细描述功能入口在哪里，如何找到...",
    "2. 操作步骤": "第一步：xxx\n第二步：xxx\n第三步：xxx...",
    "3. 结果展示": "操作完成后，系统会显示...",
    "截图": [
      {"位置": "2.操作步骤之后", "图片": "xxx.png", "说明": "xxx页面截图"}
    ]
  }
}
```
