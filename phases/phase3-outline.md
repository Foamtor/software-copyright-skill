# 阶段3：大纲生成

## 目标

生成使用说明书的三级目录大纲，供用户审阅确认。

## 流程

### 1. 生成功能模块列表

从project_info.json中提取功能模块，按用户端/管理端组织：

```
一、系统概述
  （一）系统简介
  （二）功能概览
  （三）技术架构

二、农户端功能
  （一）建房申请
  （二）办件跟踪
  （三）政策查询

三、管理端功能
  （一）审批管理
  （二）现场勘查
  （三）统计分析
```

### 2. 细化到三级目录

每个二级标题下细化操作步骤：

```
二、农户端功能
  （一）建房申请
    1. 申请入口
    2. 填写申请信息
    3. 上传附件材料
    4. 提交申请
  （二）办件跟踪
    1. 查看申请状态
    2. 查看审批进度
```

### 3. 标记截图位置

在大纲中标记需要截图的位置：

```
二、农户端功能
  （一）建房申请
    [截图: 建房申请按钮]
    1. 申请入口
    [截图: 申请信息填写页面]
    2. 填写申请信息
```

### 4. 用户审阅

展示大纲：
```
根据项目分析，生成了使用说明书大纲（三级目录），请审阅：

一、系统概述
  （一）系统简介
  （二）功能概览
  （三）技术架构

二、农户端功能
  （一）建房申请
    1. 申请入口
    2. 填写申请信息
    ...

请确认大纲，或提出修改意见（如增删章节、调整顺序）。
```

### 5. 统计截图数量

大纲确认后，统计截图标记数量：
```
大纲已确认，共需要 XX 张截图。

截图来源：
- 浏览器截取：XX 张
- 用户手动补充：XX 张
- Excalidraw生成：XX 张

请确认截图来源。
```

## 输出

- `outline.json` — 大纲结构（含截图标记）

## 注意事项

1. **必须有平台总体介绍** — 第一章必须是系统概述
2. **按功能模块组织** — 不要按技术架构组织
3. **体现页面跳转** — 操作流程要连贯
4. **截图位置要合理** — 每段操作描述后跟截图
5. **用户只审阅大纲** — 章节内容由agent自动审阅
